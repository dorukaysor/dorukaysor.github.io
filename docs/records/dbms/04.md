### **Aim**

> To study and implement various data constraints in SQL.

### **Query Syntax**

- **Create a Parent table**

  ```sql
      CREATE TABLE table_name (
        column1 datatype constraint,
        column2 datatype constraint,
        ...
      );
  ```

- **Create a Child table with multiple constraints**

  ```sql
      CREATE TABLE table_name (
        column1 datatype constraint,
        column2 datatype constraint,
        ...,
        FOREIGN KEY (column) REFERENCES ParentTable(column)
      );
  ```

- **Insert data into table**

  ```sql
      INSERT INTO table_name (column1, column2, ...)
      VALUES (value1, value2, ...);
  ```

- **Select data from table**
  ```sql
      SELECT * FROM table_name;
  ```

### **Example Script**

```sql
CREATE TABLE Department (
    DeptID INT PRIMARY KEY,              -- Uniquely identifies each department
    DeptName VARCHAR(50) NOT NULL        -- Cannot be left empty
);

CREATE TABLE Employee (
    EmpID INT PRIMARY KEY,               -- Unique ID for employee
    Name VARCHAR(50) NOT NULL,           -- Name is mandatory
    Email VARCHAR(100) UNIQUE,           -- No duplicate emails allowed
    Age INT CHECK (Age >= 18),           -- Validates that age is 18 or above
    City VARCHAR(30) DEFAULT 'Mumbai',   -- Sets 'Mumbai' if no city is provided
    DeptID INT,                          -- Foreign Key column
    FOREIGN KEY (DeptID) REFERENCES Department(DeptID) -- Links to Department table
);

INSERT INTO Department (DeptID, DeptName) VALUES
(1, 'Human Resources'),
(2, 'IT Support');

INSERT INTO Employee (EmpID, Name, Email, Age, DeptID) VALUES
(101, 'Rohan Das', 'rohan@example.com', 24, 2);

SELECT * FROM Employee;
DESCRIBE Employee;
```
